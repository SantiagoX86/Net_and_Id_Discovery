## 1. Purpose

This document records the approved M9 WP2 installation workflow decision for the Enterprise Security Discovery Orchestration Framework.

This artifact defines the determined installation workflow for controlled environment readiness and venv-based package installation.

This document is not the project README, operator guide, or end-user usage manual. Formal README and operator guidance will be handled in a later M9 work package.

---

## 2. Scope

This workflow applies to controlled installation preparation for authorized project environments.

The workflow covers:

- Obtaining approved project source
- Entering the approved project root
- Running the platform-appropriate readiness script
- Stopping and escalating if readiness fails
- Creating a local virtual environment
- Activating the virtual environment
- Installing the package into the active virtual environment
- Validating the installed console command
- Running the approved discovery command against an authorized target

This workflow does not introduce new discovery, inference, correlation, orchestration, reporting, interface, authentication, or target-interaction behavior.

---

## 3. Approved Installation Model

The approved M9 WP2 installation model is:

```text
readiness check
→ venv creation
→ venv activation
→ package installation with pip inside the active venv
→ installed command validation
→ authorized discovery execution
```
