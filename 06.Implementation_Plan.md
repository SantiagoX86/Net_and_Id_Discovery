## 1. Purpose

This Implementation Plan defines the build sequence, milestones, and verification activities for Phase 4 (Construct) of the WAUIG Bank SDLC.

---

## 2. Implementation Principles

- Maintain alignment with FRD, TRD, and ADD
- Preserve auditability and determinism
- Avoid violation of non-exploitative security constraints

---

## 3. Work Breakdown Structure (WBS)

### 3.1 Core Framework (M1)

- Core models and orchestrator implementation

---

### 3.2 Network Discovery Domain (M2)

- TCP connect-based service detection
- Predefined port scanning
- Structured findings generation

---

### 3.3 Output & Reporting (M3)

- JSON serialization
- Markdown reporting
- Evidence preservation

---

### 3.4 Identity Discovery Domain (M4)

- Implementation of identity-related service detection
- TCP connect-only probing of identity services
- Integration with orchestrator and reporting modules

---

### 3.5 Entry / Control Layer (CLI) (M5)

- CLI execution interface
- Parameter handling for targets and domains

---

## 4. Verification Plan

- Controlled lab-based validation (Kali → Windows VM)
- Deterministic re-execution testing
- Output verification (JSON and Markdown)

---

## 5. Dependencies

- Sequential milestone execution (M1 → M5)
- Completion of prior milestones required before progression

---

## 6. Risks

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Design drift | High | Enforce alignment with governing documents |
| Non-deterministic behavior | Medium | Enforce ordering and timeout controls |
| Environment variability | Low | Standardize lab configuration |
